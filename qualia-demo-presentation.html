The RICE-POT Framework — Defect Radar

7 dimensions that make every AI-generated bug ticket complete, consistent, and immediately actionable

R
Risks
What could go wrong?
Business impact, data loss risk, security exposure, regulatory impact
I
Items
What is affected?
Feature area, component, module, user journey affected by the defect
C
Criteria
How to verify the fix?
Steps to reproduce, fix acceptance criteria, test verification steps
E
Environment
Where does it happen?
Browser, OS, screen resolution, locale, test environment, device type
P
People
Who is impacted?
Affected user roles, personas, customer segments, internal teams
O
Objectives
What should happen?
Actual vs expected behaviour delta — the core bug statement
T
Tools
How to fix and track?
Fix effort estimate, root cause layer, recommended debugging tools
